# 架类图 Java 合成 — 方案核对（2026-07-14）

> 大改（碰核心生图逻辑），先对齐再动手。用户已定方向③：Java 合成根治结构变形。

## 一、为什么要做（根因）
架类走整图 AI（image2image），对细杆/镂空/多层架体脑补三维摆放 → **穿模/浮空/变形/造型件丢失**（7张全崩）。
已加的强文字约束（structLock、结构保真、image-shelf-main.txt）在正常路径上生效，**仍压不住** —— 是模型对复杂结构件的固有短板，加 prompt 收益极低。

## 二、Java 合成的本质
**不让 AI 重画架体结构**（变形根源），改用 Graphics2D 拼版式：
架体直接用白底图（whiteToTransparent 抠出）→ 结构 100% 保真、零变形。文案/标签/背景用 Java 画。

## 三、已有可复用积木（ShowerCompositor，无需新造）
- `whiteToTransparent` 白底转透明、`drawImageFit` 等比贴图
- `drawFitText`/`drawBottomBanner` 文案通栏、`drawAccCard` 功能卡
- `bgColor`/`sampleBgColor` 背景取色、`compositeMainQtyCardAt` 多件框

## 四、关键权衡（必须你拍板）
Java 合成结构不变形，但**背景和功能小图会"降级"**：
| 元素 | 整图AI(现状) | Java合成(新) |
|---|---|---|
| 架体结构 | 会变形❌ | **白底图贴入，100%保真✅** |
| 背景 | 真实场景(厨房台面/浴室墙) | **纯色/渐变棚拍风**（Java画不出真实场景） |
| 功能小图 | AI摆拍不同角度 | **白底图缩略图**（没法凭空造角度） |
| 文案标签 | AI渲染(偶尔错字) | **Java精确渲染，零错字✅** |
| 被收纳物(锅盖/衣物) | AI摆放→穿模浮空❌ | **不放**（或白底图叠放，但易假） |

**取舍**：结构保真 + 文案零错 ⟷ 背景/场景变简单。对"防比价、结构真实"是划算的；对"场景高级感"是牺牲。

## 五、拟实现（若你确认）
1. `ImageGenService.isShelf` 分支：不再调 AI，改调新方法 `compositor.compositeShelf(...)`。
2. 新方法 `ShowerCompositor.compositeShelf(白底架体图, kind, colorName, shelfMeta)`：
   - 背景：纯色/浅渐变（取白底图主色或品类默认）铺满。
   - 架体：whiteToTransparent 抠出 → drawImageFit 居中偏一侧放（占画面主体，比例真实）。
   - 文案：从 shelf-prompts.json 的品种文案里提标题+功能标签词 → drawFitText 画到左/上区。
   - 功能小图：白底架体缩略图 2-3 个放侧栏（或省略，只留文案标签）。
   - 底部：drawBottomBanner 放品类名+卖点。
3. shelf-prompts.json 复用（提取文案），或新建 shelf-layout.json 存"标题/标签/色块位置"结构化版式。
