# M11 · 生图重构：改用羽刃自定义模式（生图效果对齐羽刃）

## 背景/目标
测试反馈：本项目生图效果比羽刃差很多。目标是"选品类+主件+导入白底图 → 直接出图上新"，最大化自动化。决定：生图这部分**只用羽刃(ele-business)的自定义模式**，本项目现有生图逻辑重构对齐它。

## 差距根因（已核对代码）
**底层 GptImageAgent 是同一套**（已迁入 gofu-cloud），差距在**编排/提示词策略**：
- **本项目 FlowController.buildMainPrompt**：静态字符串模板，`"将白底图中的产品融入场景，突出卖点X"`——**从不看实际产品图**，prompt 千篇一律 → 效果差。
- **羽刃自定义模式（两步）**：
  1. `/api/custom_analyze` → `analyzeCustomImagePrompts(userPrompt, whiteImgs, count)`：**Gemini 真看白底产品图**+用户要求，扩写出**针对该产品的多段可编辑提示词**。
  2. `/api/custom_generate`：**第1张串行建基调**(ref=白底图) → **第2~N张并行**(ref=白底图+第1张)，`buildSeriesPrompt` 加系列一致性约束。auto 比例按参考图推断(resolveAutoAspect)。

关键差异：羽刃**先用视觉模型分析真实产品图再生图**，本项目直接套模板。这是效果差的主因。

## 重构方案（分块）
### A. 迁入羽刃自定义模式生图核心（云端）
- `ImageGenerationService.analyzeCustomImagePrompts` 从 ele 迁入 gofu-cloud（已有 GptImageAgent/generateImageMulti/resolveAutoAspect/buildSeriesPrompt 基础）。
- FlowController.genAllMains 重构：先 analyzeCustomImagePrompts(白底图, 品类/卖点) 得产品专属 prompt → 第1张串行基调 → 2~N 并行(ref+第1张)。替换现有 buildMainPrompt 静态模板。
- 详情图/SKU 图沿用（详情已配对，SKU 走乐羽防比价）。

### B. 重绘 + 重新生成（对齐羽刃）
- 核查本项目现状：
  - 重生：GenController `/api/gen/regenerate`(已有?) + 前端 regenImage
  - 重绘：GenController `/api/gen/inpaint`(已有,mask) + 前端 repaintImage
- 对齐羽刃 `/api/inpaint`(局部重绘,mask) + 自定义模式单张重生逻辑。需对比现状补齐。

### C. 前端布局（一键生成区不浪费位置）
- 主图张数/主图比例/方案数/防比价模式 四个下拉紧凑排列(一行 flex-wrap 紧凑，不留大空位)。

## 待确认/待查
1. 本项目重绘/重生现状 vs 羽刃——需读 GenController + workbench regenImage/repaintImage 对比后定补齐范围。
2. 自定义模式的"用户要求(prompt)"从哪来？羽刃是用户手输；本项目要自动化，应由品类/卖点/主件名自动生成这段"要求"。
3. 是否保留乐羽 SKU 防比价贴图（SKU 图效果目前 OK？），还是 SKU 也改自定义模式？—— 用户说"生图只用羽刃自定义模式"，需确认是否含 SKU。
