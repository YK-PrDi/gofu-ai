## 一、需求（来自 plan 完整愿景②）
用户导入**一整套成品图**（已做好的主图/详情）→ **换基调风格**（可多变体）→
（后续接 A/B：上多链接跑市场数据留最优，但 A/B 本次后置）。
本次风格迁移只做到"**成品图换风格、出变体**"。

## 二、已有可复用能力（调研确认，不用新造）
- **图生图**：`GptImageAgent.generateMulti(prompt, refImagePaths, ...)` — 喂成品图当参考 + "换成XX风格"prompt 即可重绘。
- **风格库 STYLE_PROMPTS**：9 种现成（科技蓝/少女粉/高级灰/自然绿/暖阳橙/卡其/淡黄/米黄/原图延展），在 FlowController。
- 羽刃**没有**独立的"风格迁移"机制，就是 styleId + 图生图组合。所以直接复用这两样即可。

## 三、风格迁移 vs 现有 styleId 的区别（关键）
| | 现有 styleId | 风格迁移(本次) |
|---|---|---|
| 输入 | 白底图（产品原图） | **成品图**（已做好的主图/详情） |
| 操作 | 从白底图 AI 从零生成 | **对成品图 image2image 重绘换基调** |
| 产物 | 全新主图 | 同内容、换风格的变体 |

## 四、待你拍板（3 个问题，定了我才写）

### 问题1：入口在哪？
- **A. 独立区块**（工作台内加「🎨 风格迁移」区块）：选图/传图 → 选风格 → 出变体预览。独立、简单、先能用。
- **B. 接进文件夹批量**：批量导入的成品图，上新前先批量换风格。跟 plan 愿景一致但更复杂。
- 建议：**先 A**（独立区块快速落地），B 等 A 跑通再接。

### 问题2：风格怎么选？出几个变体？
- **A. 用户选 1 种风格** → 对整套图重绘出 1 套变体。
- **B. 用户选多种/系统出 N 种** → 一套图出 N 个风格变体供挑（对应 plan"多变体上多链接"）。
- 建议：**先 A**（选1种出1套），多变体等 A/B 测款一起做。

### 问题3：产物去哪？
- **A. 只预览下载**（换完风格的图给用户看/存本地）。
- **B. 回写进 ProductContext**（换风格后的图直接进 visual.mainImages，能接着上新）。
- 建议：**B**（换完能直接上新才闭环）。

## 五、拟实现（若按建议 A/A/B）
- 后端：`POST /api/flow/style-transfer` { contextId 或 图路径列表, styleId } →
  对每张成品图 `generateMulti("按STYLE_PROMPTS[styleId]换基调,保持产品/构图/文案不变", refs=[该成品图])` 重绘 →
  传 COS → 回写 context.visual.mainImages（覆盖或新增变体）。
- 前端：工作台加「🎨 风格迁移」区块：选风格下拉(9种) → 「对当前商品整套图换风格」按钮 → 轮询进度 → 预览。
- 复用：STYLE_PROMPTS、generateMulti、uploadIfCos、pollTask 全现成。

## 六、已实现（A/A/B，用户确认）✅
- **后端** `POST /api/flow/style-transfer` {contextId, styleId}：对 ctx 已有主图+详情逐张 image2image
  换基调（成品图当唯一参考 + STYLE_PROMPTS[styleId] + "保内容只换风格"硬约束）→ 传COS →
  **覆盖回写** ctx.visual.mainImages/detailImages（能接着上新）。单张失败保留原图。异步任务，走 /task 轮询。
  复用：STYLE_PROMPTS、genWithRetry(图生图)、uploadIfCos、localizeWhite、flowTasks。
- **前端**：结果预览区上方「🎨 风格迁移」区块（紫色）—— 选风格(9种，排除random) → 「整套换风格」
  → 复用 pollFlowTask 轮询 → loadContext 刷新预览。/api/flow/** 经 cloudgw 转发到云端。
