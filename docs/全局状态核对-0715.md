## 十二、07.16 C组 #3 批量上新品类：文件夹名带品类（免主件库）

**方案**（用户拍板，比建主件库简单）：商品文件夹名写成 `品类-主件名`，代码解析填入，批量上新不再需要人工选品。
- 约定：第一个`-`分隔品类与主件；主件多个用`+`。例：
  - `锅盖架-落地锅盖架` → category=锅盖架, productName=落地锅盖架
  - `锅盖架-落地款+吸盘款` → 多主件保留+
  - `家装主材>厨房>厨房挂件>锅盖架-落地款` → 全路径也支持
  - 无`-`(旧命名) → category空, 退化仍需人工选品
- `SemiAutoOrchestrator.parseFolderName` 解析；`ListingService.resolveCategoryPath` 把叶子("锅盖架")解析成预设库全路径(索引就用 product-info-presets 的 key 集,不另建品类树)。
- `buildConfig` 设 category+productType+品类预设属性(材质/产地…自动填,与单品上新一致)。
- `mvn compile` SUCCESS。

⚠️ **仍需注意**：#3 反馈还提到"检测不到 SKU 图"——那是 SKU 图来源问题(靠前端反推快麦编码/选品给 skusByProduct)，本次只解决"需要选品(品类)"。SKU 图仍需按 ERP 编码命名放 sku 目录或走反推。
