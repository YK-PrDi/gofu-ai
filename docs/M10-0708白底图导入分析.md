## 回传快麦 API 已查明（open-doc.kuaimai.com）
- **method**: `item.general.addorupdate`（修改/新增普通商品）
- **图片字段**: 商品级 `picPath`(预览图URL) / SKU级 `skus[].picPath`(SKU图URL)
- **关键**: picPath 是 **URL 不是二进制**！所以回传流程必须：
  1. 导入的图先传到可公网访问的地方(COS)拿 URL
  2. 再调 addorupdate 设 picPath=该URL
  - 必填参数：`outerId`(商家编码) + `title`(商品名，必填) → 回写前要先拿到该单品现有 title(从缓存取)
- **依赖**: 回传依赖 COS 可用(欠费时无 URL 可传)；且 addorupdate **会真改快麦商品档案**(中高风险,动线上ERP数据)。

## 最终方案
**A. 导入区补齐主件+配件缺图提示 + 本地归类**（不动 ERP，安全，先做）
- autoFetchWhite 检测主件；新增配件缺图检测；右栏 missing 分「主件缺X / 配件缺Y」列出
- 导入的图按目标编码归类(entry.whites / entry.accWhites)

**B. 回传快麦**（动 ERP 数据，加确认）
- 新增 KuaimaiService.uploadItemImage(outerId, title, picUrl) → call("item.general.addorupdate", {outerId, title, picPath})
- 新增 KuaimaiController POST /api/erp/upload-white-image：入参 outerId + 本地图 → 传COS拿URL → addorupdate 回写
- 前端导入某编码的白底图后，调该端点回传；**回传前弹确认**(会改快麦商品档案)
- COS 欠费时回传跳过(无URL)，只本地用，提示用户
